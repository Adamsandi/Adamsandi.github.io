Cara Kerja Website:

-Pengguna Memasukkan URL: Proses dimulai ketika pengguna memasukkan alamat URL (Uniform Resource Locator) ke dalam browser mereka. URL adalah alamat unik yang digunakan untuk mengidentifikasi sumber daya di internet.

-Permintaan HTTP: Browser mengirimkan permintaan HTTP (Hypertext Transfer Protocol) ke server yang ditentukan oleh URL. Permintaan ini berisi informasi tentang sumber daya yang diminta, seperti halaman web atau file gambar.

-DNS Resolution: Jika URL yang dimasukkan tidak berupa alamat IP langsung, maka sistem DNS (Domain Name System) diaktifkan. DNS mengonversi nama domain ke alamat IP. Misalnya, www.contoh.com bisa diubah menjadi alamat IP tertentu.

-Permintaan Sampai ke Server Web: Setelah mendapatkan alamat IP dari DNS, permintaan dikirimkan ke server web yang sesuai.

-Web Server Menerima Permintaan: Server web menerima permintaan HTTP dan memprosesnya. Server web ini dapat menjalankan perangkat lunak khusus (seperti Apache, Nginx, atau Microsoft IIS) yang bertanggung jawab untuk mengelola permintaan ini.

-Pemrosesan Server: Server web mengambil file atau sumber daya yang diminta (misalnya, halaman HTML, gambar, atau skrip) dan memprosesnya. Jika diperlukan, server web juga dapat berkomunikasi dengan database atau server lain untuk mendapatkan data yang diperlukan.

-Mengirimkan Hasil ke Browser: Setelah pemrosesan selesai, server web mengirimkan hasilnya kembali ke browser pengguna melalui protokol HTTP.

-Rendering Halaman di Browser: Browser menerima respon dari server web dan merender halaman web untuk ditampilkan kepada pengguna. Ini melibatkan interpretasi HTML, CSS, dan eksekusi skrip jika ada.

Hubungan antara Web Server, Domain, dan Hosting:

-Domain: Ini adalah alamat yang mudah diingat yang digunakan oleh pengguna untuk mengakses suatu situs web. Domain dapat dibeli melalui registrar domain dan mewakili alamat IP tempat server web dihosting.

-Hosting: Ini adalah layanan yang menyediakan tempat untuk menyimpan file dan data situs web di server. Pemilik situs web menyewa ruang server dari penyedia hosting untuk membuat situs mereka dapat diakses secara online.

-Web Server: Ini adalah perangkat keras atau perangkat lunak yang bertanggung jawab untuk menerima dan memproses permintaan HTTP dari pengguna. Server web ini dihosting di tempat yang disebut hosting dan dapat diakses melalui domain yang terkait.
